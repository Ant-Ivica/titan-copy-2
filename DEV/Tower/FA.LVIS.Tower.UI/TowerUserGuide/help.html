  • Basic Information

  • Logging On to Tower
    • Tower is an internal application that utilizes the First American User's Network credentials for access; thus, no further login is required to gain access to Tower once user is logged into the First American Network.

      •  

    • To use Tower, user must first be set up with a profile in Tower so the system will have the necessary information to manage the user's access to specific screens and information.
  • Access Rights and Tenancy
    • Tower users are set up with different access rights; therefore, it is possible for two Tower Users to see slight variations on their screens. It is not advisable to share your Tower URL with another Tower User as access rights conflict may exist.

      •  

    • There are three levels of user access: Super Admin, Admin, and Standard User. Access rights are assigned based on the User's role and responsibility. A Super Admin user will have greater access than an Admin or a Standard User.

      •  

    • Every Tower User belongs to a specific Tenant (e.g. Agency, LVIS, Mortgage Solutions, etc...). Tower uses this tenant concept to restrict the viewing and accessing of information from one tenant type to another.

  • Main Menu

  • Home
    • This is Tower's Landing Page, the default view when Tower is launched. The User ID and Tenancy are statically displayed in the title bar. The activities of the Business Exception Queues and Technical Exception Queues are shown in this view.

      •  

    • A color coded "Application Status" indicator will always display on the upper right hand corner in every screen. This shows the health of the connection between Tower and the various applications (e.g. RealEC, FAST, etc...)"Green" = Good. "Red" = Not Good. Clicking on the droplist and hovering over the "Red" application will display the reason why there is an issue.

      •  

    • "Tiles" are used to represent each exception type. Each tile is labeled with the number of unreolved exceptions and the date and time of the last entry. (The date and time of the last refresh will be used if there is "0" exception activity).

      •  

    • Access to the Exception Type's Grid View may be obtained by clicking on the tile.

      •  

    • Each bar graph represents an hour of activities. Hovering over a bar graph will display a breakdown of exception activities for that hour. Exception activities may be selectively omitted from the graph by clicking on the activity type label in the lower section of the graph as shown above.
  • Reporting
    • This menu item allows users to view a summary of all orders activities, including the incoming and outgoing messages.

      •  

    • Orders may be searched by "Date Range" or "Reference Number". For "Date Range", User can specify the Start and End Dates. "Reference Number" type may be selected from the drop list.

      •  

    • Key order information may be viewed by clicking on the "+" symbol on the leftmost side of the Grid Line Item.

      •  

       

    • Double clicking on the Grid Line Item will display the "Order Information" as well as the "Message Flow" activities going in and out of LVIS. (Attachments in messages are marked with a "paperclip" icon. Clicking on the "paperclip" icon will open up the XML for review.).

      •  

    • A "Green" arrow indicates a smooth flow in or out of LVIS with no issues. A "Red" arrow indicates an issue has occurred. Clicking on the "Red" arrow will provide an "Exception Description" pop up with additional information.

  • Grid UI & Controls

  • Grid UI & Controls
    • The Grid View is common to many of the screens in Tower; thus, the controls and its usage are the same and should be very easy to master as they are similar from one screen to the next.

      •  

    • Delete   -    To delete a record. Double click on the record to open up the details view. If a record is "Delete-able", a "Delete" button will be available in the bottom right area in the view. Not all records are "delete-able". The user's access rights will also determine whether or not a record is "delete-able".

      •  

    • Edit   -    Double clicking on a grid row will open up the details view for that record. Not all records are editable. System will gray out fields that are Read-Only. The user's access rights will also determine whether or not a field is editable.

      •  

    • Export   -    An "Export" function is available in drop list located in the the upper right corner of the grid. Two export formats are available: pdf and csv. User may elect to: 1) Export all data, 2) Export visible data, 3) Export selected data.

      •  

    • Filter   -    Each column heading section contains a textbbox where a filter value may be entered. System will dynamically display the matched results in the column. A "Clear All Filter" function is available in the drop list located in the upper right corner of the grid.

      •  

    • Group   -    A "Group" function is available in the drop down list located in the upper right corner of each column heading section. The first "Grouped" items will be moved to the leftmost column and displayed in an ascending order. Subsequently "Grouped" items will appear to the right of the last group, if one exists; if not, it will assume the leftmost column position. An "Ungroup" function is available for a grouped column. Once "ungrouped", the items will return to its default column location. Note: To quickly ungroup everything and reset back to default settings, click on another view and return to your present location.

      •  

    • Hide Column   -    Grid columns may be selectively hidden from display by: 1) Selecting the "Hide Column" option in the drop list in the column heading, or 2) Check or uncheck the list of columns displayed in the drop list in the upper right corner of the grid. To "Unhide" a column, change the "X" to a check mark in the drop list located in the upper right hand corner of the grid. Note: To quickly unhide all columns and reset back to default settings, click on another view and return to your present location.

      •  

    • Pagination   -    A pagination control is available on the bottom left corner of the grid.

      •  

    • Sort   -   "Sort Ascending" and "Sort Descending" functions are available in the drop down list located in the upper right corner of each column heading section. Sort ascending/descending may also be toggled back and forth by clicking on the Column Heading Label. Note: To quickly "Unsort" everything and reset back to default settings, click on another view and return to your present location.

  • What web browsers are supported?
    • Most web browsers will work with Tower; however, the best user experience will be attained with Google Chrome or Internet Explorer. A warning message will be displayed to the user when Tower is launched with a web browser other than Chrome or IE.
  • How do I add a Tower user?
    • Tower is an internal application that utilizes the First American User's Network credentials for access; therefore, a prospective user must have First American Network Credentials (a.k.a. Network Login and Password) in order to access Tower. A Super Admin can add a user by setting up a profile for that user in the "Add New User" screen in the Security main menu.
  • How do I delete a Tower user?
    • Tower does not allow a user to be deleted; however, a user can be made "Inactive" so access to Tower is denied. An Admin or Super Admin can inactivate a user by unchecking the "Active" checkbox in his/her user profile in the "Edit User Profile" view in the Security main menu.

  • How do I change the user's access rights?
    • An Admin or Super Admin can access the "Edit User Profile" view from the Security main menu to change a user's access right. However, the access right can only be changed laterally or downward based on the access level of the person performing the change. For example, an "Admin" cannot change a "Standard User" to a "Super Admin". But, an "Admin" can change a "Standard User" to an "Admin". Similarly, an "Admin" and downgrade a fellow "Admin" to a "Standard User".
